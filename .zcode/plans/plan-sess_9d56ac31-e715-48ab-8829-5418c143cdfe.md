## Plan: Vulkan 2D/HUD/Text Draw Path

### Goal
Make the menu, HUD, and text visible under `--render vulkan` by implementing the 2D screen-space draw path that is currently inherited as no-ops from `EngineDummy`.

### Architecture (mirrors GL33, adapted to Vulkan's deferred command recording)

The GL33 2D path works like this: each `Draw2D`/`DrawPoly`/`DrawLine` call builds a `TLVertex[4]` quad (pos.xyz, rhw, color, specular, uv0, uv1 — 40 bytes), accumulates it into a CPU vertex mirror keyed by texture, and flushes per-texture with one `glDrawElements`. The "projection" is pre-transformed rhw vertices: a 2-vector `vpScale = {2/width, 2/height}` scale baked into the vertex shader, no matrix.

The Vulkan backend can't emit draws inline (the command buffer isn't being recorded during game draw calls — it's recorded later in `RecordBootstrapCommand` during `NextFrame`). So the Vulkan pattern must: **buffer** 2D primitives during the game's `Draw2D` calls, then **emit** them in `RecordBootstrapCommand` after the 3D scene block.

### Implementation steps

**1. New shaders: `screen.vert.glsl` + `screen.frag.glsl`**
Port `vsScreen.glsl`/`psNormal.glsl` (2D subset) to Vulkan GLSL:
- Vertex: `gl_Position = vec4((pos.x*vpScale.x - 1.0)*w, (1.0 - pos.y*vpScale.y)*w, pos.z*w, w)` where `w = 1.0/rhw`. `vpScale` comes from a push constant `{2/width, 2/height}`. Inputs: pos.xyz, rhw, color(BGRA8 unorm), uv0 (matches TLVertex layout, minus specular/t1 which the minimal 2D path doesn't need).
- Fragment: `outColor = vColor * texture(tex0, vUV0)` with alpha-test support. Simplified from GL33's psNormal (drops shadows/fog/night-eye for the first slice — those can come later).

**2. New headers**
- `ScreenPushConstantsVK.hpp` — push constant struct `{ float vpScale[2]; }` (16 bytes, with padding to 16-align for the push_constant block minimum).
- `ScreenVertexLayoutVK.hpp` / extend `VertexLayoutVK.hpp` — vertex input descriptions for the 2D TLVertex layout (pos vec3 @0, rhw float @1, color BGRA8 @2, uv0 vec2 @3).

**3. EngineVK additions (EngineVK.hpp / EngineVK.cpp)**

*State members:*
- `VkPipeline _screenPipeline` + `VkPipelineLayout _screenPipelineLayout` + shader modules
- `VkDescriptorSetLayout _screenDescriptorSetLayout` — one combined image sampler at binding 0 (reuse the existing `_textureDescriptorSetLayout` pattern)
- `vk::BufferVK _screenVertexBuffer` — persistent host-visible vertex buffer for 2D vertices
- `std::vector<TLVertex> _screenVertices` — CPU-side vertex accumulation (cleared each frame in `InitDraw`)
- `std::vector<uint16_t> _screenIndices` — CPU-side index accumulation

*2D draw overrides (override the EngineDummy no-ops):*
- `Draw2D(const Draw2DPars&, const Rect2DAbs&, const Rect2DAbs&)` — build a TLVertex[4] quad (port the exact vertex-building math from `EngineGL33_2D.cpp:62-100`), resolve the texture from `pars.mip._texture` via `GetOrCreateTextureResourceId`, append 4 vertices + 6 indices to the CPU buffers, record texture id.
- `DrawPoly(const MipInfo&, const Vertex2DAbs*, int, ...)` and the `Vertex2DPixel` overload — fan-triangulate into the same buffers.
- `DrawLine(const Line2DAbs&, ...)` — port the line-to-quad math from `EngineGL33_2D.cpp:112+`.

*Lifecycle:*
- `InitDraw()` — clear `_screenVertices`/`_screenIndices` (alongside the existing `_drawItems.clear()`).
- `CreateScreenPipeline()` / `CreateScreenPipelineLayout()` / `CreateScreenDescriptorLayout()` — build the 2D pipeline (depth disabled, alpha blend enabled, matching the GL33 `ScreenSpace` pass state). Called during init alongside the scene pipeline.
- `EnsureScreenVertexBufferCapacity()` — grow the vertex buffer if `_screenVertices` exceeds capacity (follow the `EnsureDrawConstantsBufferCapacity` pattern).
- Destroy methods for the 2D resources.

*Emit (in `RecordBootstrapCommand`, after the scene block, before `vkCmdEndRenderPass`):*
- Upload `_screenVertices`/`_screenIndices` to the GPU buffers via `UploadMappedBuffer`.
- Bind `_screenPipeline`, bind the 2D vertex buffer, push the `vpScale` constant `{2.0f/_width, 2.0f/_height}`.
- For each batch of vertices sharing a texture, bind that texture's descriptor set and issue `vkCmdDrawIndexed` over the corresponding index range. (First implementation: one draw per recorded texture-change boundary. Can batch later.)

**4. CMakeLists.txt** — add the two new `.glsl` shader sources to the shader-embedding list for `PoseidonVK`.

**5. Tests** — extend `test_vulkan_scene_shaders.cpp` (or a new `test_vulkan_screen_shaders.cpp`):
- screen.vert/frag compile under Vulkan GLSL rules
- push constant struct layout matches shader contract
- vertex input descriptions match TLVertex layout

### What this enables
After this work, `--render vulkan` will render the main menu, in-game HUD, and text. Combined with the projection fix already done, a mission will show terrain + models (3D) overlaid with HUD/text (2D). This is the core "Render terrain, models, sky, water, cockpit, HUD, text" Phase 2 roadmap item — the 2D half.

### What this does NOT yet do
- Sky/water/shadow passes (separate 3D passes, later)
- Per-fragment lighting parity / fog / night-eye on 2D (simplified fragment shader for now)
- Batching optimization (one draw per texture-change is fine for correctness; can optimize to instanced/batched later)
